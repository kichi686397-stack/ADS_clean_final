# ADS 投放与客户信息工作台（整理版）

## 启动
1. 打开 `token.txt`，粘贴 Meta Token。
2. 双击 `install.bat` 安装依赖。
3. 双击 `frontend.bat` 打开前端。

网页顶部包含三个入口：

- `批量投放`：素材预览、投放队列、投放前检查、生成计划并运行。
- `客户信息`：按产品增量同步、全量同步或指定日期导出，查看日志并下载生成的 Excel。
- `广告数据`：读取最近 7/14/30 天广告组表现，按广告系列（产品）和国家筛选；点击花费、展示、点击、Leads 或单条线索成本卡片可直接切换对应折线图，并可在广告系列汇总和明细表之间切换、下载 Excel。

## 季度预算自动计算

网页顶部提供独立的“预算中心”页面，支持导入 `.xlsx`、`.xlsm` 或 `.csv`。预算表至少需要“产品（或 Campaign ID）”“国家/地区”和“初始预算”；“语种”是可选列，填写后会自动带入该地区的一键投放，留空则使用素材默认语种。年份和季度为空时默认当前季度，也可以先下载模板填写。系统会按产品对应 Campaign 检查该地区是否已有 ACTIVE 广告组。

每日预算输入框可以留空。投放前检查和生成计划时会按以下公式匹配产品对应 Campaign：

```text
自动日预算 = max(0, 季度初始预算 - Campaign 本季度已花费) / 本季度剩余天数（包含今天）
```

季度已花费使用一小时本地缓存；手动填写每日预算时手动值优先。未匹配预算、未匹配 Campaign 或季度预算已用完时，投放前检查会警告，生成计划会阻止无有效预算的广告组。

预算看板会标记“已有 ACTIVE 广告”或“缺投放”。缺投放地区可打开“一键投放”弹窗，默认全选并支持全选/清空；投放语种锁定为预算表填写的固定语种，弹窗中不再重复选择。同一产品可以统一选择一次卖点，系统只显示支持对应预算语种的素材；没有匹配语种素材的预算项会自动取消选择并标记。之后可以先加入队列检查，也可以选择 Dry Run 或 LIVE，在完整投放前检查通过后直接生成计划并运行。

广告数据导出的 Excel 包含 `广告组明细`、`每日汇总`、`广告系列汇总` 和 `环比汇总` 四个工作表；选择某个广告系列或国家后，页面指标、图表、表格和下载文件都会同步过滤，广告组明细会保留国家列。
网页明细表按“日期、国家、广告组”展示，不重复显示 Campaign；最后一列会将当前行与上一周期对应日期、同国家、同广告组的数据匹配，展示花费、Leads 和 CPL 环比。

广告数据支持同周期环比：最近 7 天对比前 7 天、14 天对比前 14 天、30 天对比前 30 天。KPI 显示环比升降，折线图用虚线展示上一周期，广告系列表展示花费、Leads 和线索成本环比；Excel 另含 `环比汇总` 工作表。

广告数据默认使用 60 分钟本地缓存，并写入 `_cache/insights_data.json`；重复打开网页、切换模块或重启本地服务时都会优先读取缓存。缓存超过一小时后页面会自动更新，只有点击“强制刷新”才会跳过缓存立即请求 Meta。

## 自动化规则中心

`广告数据` 页面底部提供自动化规则中心。默认未启用，首次启用需要确认；启用后，本地工作台运行期间每 6 小时检查一次：

- 当前广告组最近 7 天的地区 CPL 比该账户上一个自然月同地区整体 CPL 高 25%：提示更换素材。
- 高 50%：自动把广告组状态改为 `PAUSED`。
- 连续 3 天有花费但 0 Leads：自动改为 `PAUSED`。
- 今日花费超过前 7 天日均的 1.5 倍、达到 $20 且 0 Leads：自动止损暂停。
- 近 3 天至少 5 Leads，且 CPL 比地区上月基准低 20%：日预算上调 10%。
- 季度预算进度超过 110% 时日预算下调 10%；进度低于 80% 且 CPL 健康时上调 10%。
- ACTIVE 超过 24 小时仍无消耗：提示检查审核、受众、出价和支付状态，不自动修改。
- 检查产品落地页、留资表单和预算表固定语种对应的素材包，异常只提示，不自动停投。

新广告默认保护 72 小时，保护期间只展示提醒，不参与停投、扩量和预算调速。CPL、连续无留资、扩量和预算调速默认使用截至昨天的完整数据，避免当天 Leads 回传延迟；当天数据只用于花费突增观察。

自动化提供三种模式：`仅提醒`、`待人工确认`、`全自动执行`，默认是待人工确认。所有预算调整单次最多 10%，同一广告组调整后冷却 24 小时。可以先使用“只读预览”，不会修改 Meta 广告；支持勾选多项后批量执行，或只执行某一条。

每次真实暂停或调预算都会保存原状态，可在规则表或“自动化效果复盘”里一键撤销。复盘会对比执行前 3 天与执行后 3/7 天的花费、Leads 和 CPL，并统计改善、恶化、观察中以及停投的估算节省金额。旧版本操作记录没有操作编号，因此只支持查看，不能自动撤销。

命中关闭或换素材的项目还提供“换卖点重新创建”，会保留原 Campaign、国家和日预算，把新卖点加入批量投放队列；仍建议先做投放前检查。自动执行历史保存在 `日志/automation_actions.json`，预算冷却状态保存在 `config/automation_action_state.json`。

客户信息默认增量同步；全部产品不勾选时会同步全部。生成的文件仍保存在 `lead_sync/`，原有 `同步客户留言.bat` 也可以继续使用。

## 文件夹结构

```text
ADS/
├─ app.py                 # 本地前端服务
├─ run.py                 # 正式创建广告的执行程序
├─ plan.xlsx              # 生成/检查用计划表，Campaign 对照仍在这里
├─ token.txt              # 每个人本地自己的 Token
├─ frontend.bat           # 打开前端
├─ run.bat                # 手动运行 run.py
├─ install.bat            # 安装依赖
├─ scripts/               # PowerShell 启动脚本
├─ config/                # 产品线、官网URL、受众配置
│  ├─ product_settings.json
│  ├─ product_urls.json
│  └─ audiences.json
├─ _cache/                # 本地缓存：表单库、图片hash、创意缓存、受众缓存
├─ 素材包/                # 唯一素材目录：产品/卖点/语种/文案+图片
└─ 日志/                  # 运行日志
```

## 预算预警、素材疲劳与经营报表

- 季度预算看板会显示当前应花金额和实际花费的比例：高于应花 110% 标为“超进度”，季度已运行至少 7 天且低于应花 80% 标为“进度偏慢”。
- 看板根据本季度日均花费预测季末花费、预计预算完成率和预算耗尽日期，并同时展示 Leads、CPL、CTR 与频次。
- 广告数据页的“素材疲劳检测”以广告组 + 国家为单位：近 3 日花费达到 `max($20, 前3日CPL)` 后，CPL 比前 3 日恶化 25% 时提示、恶化 50% 时标为高风险；近 3 日零留资时，用已花金额作为当前 CPL 下限参与判断，避免小额波动误报。
- “预算中心”顶部的“一键生成并下载周报 / 月报”会生成 Excel，包含产品经营看板、广告系列汇总及环比、每日趋势、素材疲劳、自动化记录和广告组明细。
- 投放前检查会拦截同一 Campaign、相同卖点、国家重叠的 ACTIVE 广告组；同一批队列里的重复项也会被拦截。正式生成计划时会再次检查，避免跳过预检后重复投放。

## 文件夹整理说明

- `素材包/` 是唯一的素材入口，图片请放在 `素材包/产品/卖点/语种/图片/`。
- 文案文件名不再受限：在对应语种目录放任意 `.txt` 后刷新素材包即可自动识别。一个 TXT 可以用“正文 / 标题 / 描述”作为分段标题；也可以分别放 `正文.txt`、`标题.txt`、`描述.txt`，系统会自动合并。完全没有分段标签的普通 TXT 默认作为正文，文件名包含“标题/title/headline”或“描述/desc/description”时会自动识别对应类型。
- TXT 每个非空行会作为一条文案；使用空行分隔时，每个段落会合并成一条文案。前端预览会显示本次识别到的 TXT 文件名，正式投放脚本使用相同规则。
- 旧版根目录 `images/` 已删除，代码也不再默认读取它，避免同事误放素材。
- `config/` 放产品线、受众、官网 URL；`_cache/` 放表单库、图片 hash、受众缓存；`日志/` 放运行日志。
